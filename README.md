# Assignment: Git & GitHub Fundamentals

**Course:** DevTools  
**Submission:** Push your completed work to your GitHub Classroom repository  
**Pairs:** This assignment is completed in pairs — one student is **Partner A**, the other is **Partner B**

---

## Overview

By the end of this assignment you will:

- Have a GitHub account tied to your academic email
- Know how to clone, add, commit, push, and pull using Git
- Have collaborated with a partner through a shared repository
- Understand how Git stores data internally (the `.git` directory)

## Tips

- Run `git status` constantly — it always tells you what to do next.
- Never force-push (`git push --force`) on a shared branch.
- Commit messages should describe **why**, not just what. `"fix bug"` is bad; `"Fix off-by-one in loop counter"` is good.
- If you get stuck, `git log --oneline --graph --all` gives you a visual picture of the branch history.
